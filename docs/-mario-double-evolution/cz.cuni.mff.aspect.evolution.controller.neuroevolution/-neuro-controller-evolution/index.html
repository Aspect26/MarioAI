NeuroControllerEvolution

class NeuroControllerEvolution : ChartedJeneticsEvolution<MarioController>, ControllerEvolution

Implementation of a Super Mario controller evolution using simple neuroevolution algorithm, which evolves only weights on a given non-changing ANN. The implementation is highly customizable in terms of multiple properties of the evolution, which can be specified via primary constructor.

Parameters

controllerNetworkSettings - specifies network settings.

generationsCount - number specifying how many generations the evolution should run for.

populationSize - size of a population.

evaluateOnLevelsCount - number of levels on which each individual should be evaluated. If the individuals are evaluated on multiple level generators, this number is evenly split amongst them. If that it not possible, some generators will generate one level less.

fitnessFunction - specifies fitness function.

objectiveFunction - specifies objective function.

parallel - specifies, whether the evolution should run on multiple CPU cores.

alterers - specifies alterers (mutators) for the evolution.

survivorsSelector - specifies selector for the survivors in the evolution.

offspringSelector - specifies selector for the offsprings in the evolution.

weightsRange - specifies range for the weights.

alwaysReevaluate - specifies, whether the individuals should be reevaluated in each generation during the evolution, or only when they are created. This is useful when the fitness computation is randomized somehow.

displayChart - specifies, whether a real-time chart, which displays fitness values and objective values, should be displayed.

chartLabel - label of the chart.
